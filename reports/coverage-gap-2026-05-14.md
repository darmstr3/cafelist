## Method
For each (city, neighborhood) extracted from parsed intent we compute:
- `query_count` ‚Äî demand signal.
- `avg_quality_score` ‚Äî evaluator output, 0‚Äì10.
- `thin_result_rate` ‚Äî fraction of runs returning fewer than 3 picks.
- `dominant_failure_mode` ‚Äî most common non-`ok` failure mode.

```
priority_score = log10(1 + queryCount)
               * max(0, 10 - avgQuality)
               * (1 + thinResultRate)
               * 100    // weight vs hardcoded seeds (seeds cluster 80‚Äì100)
```

Missing `avgQuality` is treated as max gap (10) ‚Äî a run with no
evaluator output is itself a strong signal something's off.
